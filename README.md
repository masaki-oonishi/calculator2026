# Calculator

本プロジェクトは、Androidプラットフォーム向けに開発された、柔軟な計算エンジンと堅牢な状態管理機構を備えた高機能電卓アプリケーションです。
単なる計算機能の実装にとどまらず、オブジェクト指向設計における主要なデザインパターン（Strategyパターン、Mementoパターン）および厳格なカプセル化の有効性を実証することを目的として設計されています。

## 🛠️ アーキテクチャの特徴（Architectural Specifications）

### 1. ハイブリッド計算エンジン（Strategy パターンの適用）
数式の解析および評価を行うコアロジックをインターフェース化し、実行時に動的な切り替えを可能にする疎結合な構造を採用しています。
* **AST (Abstract Syntax Tree) モード**: 人間の認知構造に適合した中置記法（Infix）の数式を、階層的な木構造として解析します。構造の抽象化により、将来的な演算子や関数の拡張に対して高い保守性を備えています。
* **RPN (Reverse Polish Notation) モード**: スタック領域の操作に基づく、古典的かつ確定的な評価を行う計算エンジンです。

### 2. 高精度不変数値オブジェクト（MyNumber）
浮動小数点演算における丸め誤差やプリミティブ型の限界を排除するため、内部数値とスケール（小数点以下の桁数）を包括的に管理する値オブジェクト（Value Object）`MyNumber` を独自に実装しています。
* **完全な不変性（Immutability）**: すべての算術演算は新しいインスタンスを生成して返すため、副作用を排除し、状態の安全性を担保します。
* **値ゼロにおけるスケール正規化処理**: 演算結果またはパースされた値が `0` となる場合、内部スケールを自動的に `0` へ正規化し、不要な冗長表示（例: `0.000000000`）を防止します。
* **標準精度（9桁）への丸め込み**: 無限小数の循環時や極小値の発生時において、規定の境界で安全に演算を打ち切るガードロジックを実装しています。

### 3. 履歴管理と定数計算における数式の完全復元（Memento パターンの適用）
電卓特有の複雑な挙動である「連続する等号入力による定数計算」と「過去の状態復元」の整合性を完全に維持しています。
* **数式の動的完全復元**: 定数計算時（例: `40/8` の実行後に `=` を連打するケース）、履歴データ（Memento）には単なる演算子の断片（`/8`）ではなく、構文解析器（Parser）が再解釈可能な完全な数式（`5.000000000/8`）として結合・復元されて保存されます。
* **最小権限の原則に基づくカプセル化**: プレゼンテーション層（View）に対する不必要なパブリックセッター（Setter）を排除し、状態遷移の秩序をViewModel内部に完全に隠蔽しています。

### 4. 例外駆動設計と堅牢なUIライフサイクル連動
ゼロ除算、構文エラー、および予期せぬ実行時例外を各レイヤー間で安全に伝播・ハンドリングします。
* ViewModel内の `try-catch-finally` 構造により、異常系の発生時においても内部状態の初期化とエラーログ（履歴）の保存を確実に実行します。
* View（MainActivity）側では、一元化された `finally { updateDisplay() }` ブロックにより、いかなる例外発生時においても画面表示の整合性と同期を維持します。

## 🧪 テスト構成（Unit Tests）

ロジックの正当性とデグレード（先祖返り）の防止を担保するため、ローカルユニットテスト（JUnit 4）を網羅的に配備しています。

### 単体テストの実行手順
Android Studio内のターミナル、または任意のコマンドラインインターフェースから以下のタスクを実行することで、全レイヤーのテストが網羅的に実行されます。

```bash
# Windows環境
gradlew test

# Mac / Linux環境
./gradlew test
